# zen-Babel for macOS 0.8.2 修正リスト

## 不具合修正

- 空白の文が翻訳対象として登録される問題を修正
  - 文間の空行に対して、翻訳不能のオレンジマーカーが出ることがありました。
  - 文字を含まないセグメントは生成しないように修正しました。
  - 既存の改行/空白レイアウトは joiner 側で保持し、本文の再構成互換性を維持しています。

- 言語メニューの Source 判定タイミングを改善
  - 言語メニューを開いた時点、Paste直後、キーボードを閉じた時点で Source 言語を再判定するようにしました。
  - 判定結果に応じて、言語メニューの状態（利用可否・ヒント）を再構築するようにしました。

- Source 言語とターゲット言語が一致した場合の自動切り替えを追加
  - 言語メニュー再構築時に、判定された Source 言語が現在のターゲット言語と同一だった場合、
    直前の Source 言語をターゲットへ自動設定するようにしました。
  - 例: `en -> ja` の翻訳後に日本語を入力した場合、ターゲットを `en` に戻します。

- macOS バンドル名/成果物名の表記を `zen-Babel` に統一
  - `.app` / `.zip` / `.dmg` の命名を `zen-Babel` ベースに揃えました。
  - 実行バイナリ名・Bundle ID・URL スキーム（`prebabellens://`）は互換性のため維持しています。
  - Issue: #2
## 補足

- 翻訳、ファイルの読み込み、画像からのOCRは引き続きオンデバイスで実行されます。
