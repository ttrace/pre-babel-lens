# Pre-Babel Lens v0.6.3 (Draft)

## User-facing update / ユーザー向けアップデート

In this release, when Apple Intelligence safety guardrails are triggered, the app now supports partial translation recovery using the classic Apple Translation Frameworks. Recovered partial translations are shown with marker colors so users can recognize they were translated by a different mechanism.
このリリースでは、Apple Intelligenceの安全ガードレールが発動した際に、従来型のApple Translation Frameworksで部分翻訳を支援する機能を実装しました。部分翻訳にはマーカー色がつくので、異なる機能による翻訳であることを認識できます。

## Highlights

- On-device Translation Framework fallback on iOS
  - When Apple Intelligence returns an unsafe block for a segment, the app now attempts Translation Framework recovery first.
  - Apple Intelligence でセグメントが unsafe 判定になった場合、まず Translation Framework による補完を試みます。
- Source fallback is preserved for reliability
  - If Translation Framework also cannot translate that segment, the app still inserts source text to avoid losing content.
  - Translation Framework でも翻訳できない場合は、内容欠落を防ぐため従来どおり原文を挿入します。
- Clearer marker colors for fallback outcomes
  - Segments recovered by Translation Framework are shown in light blue.
  - Segments that fell back to source text keep the existing warning color.
  - Translation Framework で補完できた箇所は水色で表示されます。
  - 原文フォールバックになった箇所は従来の警告色のままです。

## App Store "What’s New" short copy

- EN: Improved iPhone unsafe fallback flow. The app now tries on-device Translation Framework recovery first, keeps source fallback when needed, and uses clearer marker colors.
- JA: iPhone の unsafe フォールバックを改善。on-device Translation Framework 補完を優先し、必要時は原文フォールバックを維持。マーカー色もより分かりやすくしました。

## Notes

- Translation processing is on-device.
- Some language resources may require an initial system-managed download.
- 翻訳処理は on-device で行われます。
- 一部言語では初回にシステム管理の言語データダウンロードが必要です。
