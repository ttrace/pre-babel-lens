# zen-Babel for macOS 0.9.0 修正リスト

## 不具合修正

- 空白の文が翻訳対象として登録される問題を修正
  - 文間の空行に対して、翻訳不能のオレンジマーカーが出ることがありました。
  - 文字を含まないセグメントは生成しないように修正しました。
  - 既存の改行/空白レイアウトは joiner 側で保持し、本文の再構成互換性を維持しています。

- 言語メニューの Source 判定タイミングを改善
  - 言語メニューを開いた時点、Paste直後、キーボードを閉じた時点で Source 言語を再判定するようにしました。
  - 判定結果に応じて、言語メニューの状態（利用可否・ヒント）を再構築するようにしました。

- ターゲット言語の自動切り替えを追加
  - 言語メニューを再構築するときに元言語が現在のターゲット言語と同一だった場合、
    直前の Source 言語をターゲットへ自動設定するようにしました。
  - 例: `日本語 -> 英語` の翻訳している時にSourceに英語を入力すると、ターゲットを `英語` に切り替えます。

- macOS バンドル名/成果物名の表記を `zen-Babel` に統一
  - `.app` / `.zip` / `.dmg` の命名を `zen-Babel` ベースに揃えました。
  - 実行バイナリ名・Bundle ID・URL スキーム（`prebabellens://`）は互換性のため維持しています。
  - Issue: #2

- 繁体字の中国語や英国風の英語を端末で使っている方のAI翻訳では、繁体字、英国風のAI翻訳を行います。
  - 中国語ターゲット時、Preferred Language で繁体字系（`zh-Hant` / `zh-TW` / `zh-HK` / `zh-MO`）が優先の場合、
    `Translate into Traditional Chinese script.` を付与します。
  - 英語ターゲット時、Preferred Language が `en-GB`（`en-UK` 含む）優先の場合、
    `Use British English style.` を付与します。

- iOSのローカライズを強化
  - 対応言語: `en`（英語）, `ja`（日本語）, `ko`（韓国語）, `zh-Hans`（簡体字中国語）, `zh-Hant`（繁体字中国語）
  - メニュー（`File` / `Album` / 翻訳モード切替）が端末言語で表示されるようにしました。
  - ステータス通知・同一言語ペアのエラー文言を、フォールバック文言も含めてローカライズしました。

- PDF取り込みの改行連結で、スマートクオートを含むケースを修正
  - `.\n"\n` だけでなく、`.\n”\n` のようなスマートクオート（閉じ引用符）を含む行でも、
    文末側の改行を連結するようにしました。
  - これにより、引用文が行末改行で不自然に分断されるケースを軽減しています。


## 補足

- 翻訳、ファイルの読み込み、画像からのOCRは引き続きオンデバイスで実行されます。
